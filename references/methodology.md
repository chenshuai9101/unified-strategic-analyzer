# 麦肯锡问题分析方法论 — 完整参考手册

## 一、MECE 原则

### 定义
Mutually Exclusive, Collectively Exhaustive（相互独立，完全穷尽）

### 核心要求
- **相互独立 (ME)**：每个分类之间不重叠、不交叉。任何一项只能属于一个分类。
- **完全穷尽 (CE)**：所有分类合起来覆盖全部可能，没有遗漏。

### 常见错误
| 错误类型 | 例子 | 纠正 |
|:--|:--|:--|
| 重叠 (非ME) | "国内用户 / 付费用户" | 付费用户可能在国内，用矩阵：付费×区域 |
| 遗漏 (非CE) | "大客户 / 小客户" | 缺中等客户，改为按收入分段 |
| 维度混乱 | "年轻人 / 高收入 / 常网购" | 三个维度混了，各自独立分类 |

### MECE 拆解方法
1. **二分法**: 是/否，内部/外部，线上/线下
2. **流程法**: 按业务流程步骤：获客→转化→留存→变现
3. **要素法**: 按组成要素：人、财、物、信息、技术
4. **公式法**: 按数学公式：收入=客单价×客户数×复购率
5. **矩阵法**: 两个维度交叉：紧急×重要 = 四象限

### 应用场景
- 分析问题：把大的模糊问题拆解成小的可分析的问题
- 工作规划：确保工作不重不漏
- 信息分类：确保信息完整不重复

## 二、金字塔原理

### 基本原则
1. **结论先行**：先说结论，再说论据
2. **以上统下**：每一层都是下一层的总结归纳
3. **归类分组**：每组内的思想属于同一逻辑范畴
4. **逻辑递进**：每组中的思想按逻辑顺序排列

### 结构示意
```
          ┌───── 核心结论 ─────┐
          │                    │
    ┌─────┴─────┐        ┌─────┴─────┐
    │  论据 A    │        │  论据 B    │
    └──┬──────┬──┘        └──┬──────┬──┘
  ┌────┴──┐ ┌─┴───┐    ┌───┴──┐ ┌─┴───┐
  │数据A1│ │数据A2│    │数据B1│ │数据B2│
  └──────┘ └─────┘    └──────┘ └─────┘
```

### 纵向关系（疑问/回答）
- 上层提出观点 → 下层提供支撑
- 读者自然产生的疑问 = 下一层要回答的问题

### 横向关系（归纳/演绎）
- **演绎推理**：大前提→小前提→结论（从一般到特殊）
- **归纳推理**：多个事实→共同点→结论（从特殊到一般）

### 序言结构 (SCQA)
- **S**ituation（情境）：描述已知的稳定状态
- **C**omplication（冲突）：打破稳定状态的变化
- **Q**uestion（问题）：由此产生的需要回答的问题
- **A**nswer（答案）：核心建议/结论

## 三、假设驱动

### 核心理念
不追求完美分析，先提出初始假设，用最小数据验证，快速迭代。

### 流程
```
提出初始假设 → 设计验证方案 → 快速收集数据 → 
验证/证伪假设 → 修正假设 → 深入验证 → 形成结论
```

### 假设构建原则
1. 基于已有知识和经验
2. 大胆假设，小心求证
3. 保持多个备选假设
4. 优先验证最关键假设

### 假设检验方法
| 方法 | 适用场景 |
|:--|:--|
| 数据分析 | 有内部数据 |
| 专家访谈 | 缺少直接数据 |
| 案例类比 | 跨行业参考 |
| 快速试验 | 可做小范围测试 |
| 逻辑推演 | 无数据可获取 |

## 四、议题树 (Issue Tree)

### 三种议题树

#### 1. 为什么树 (Why Tree / Diagnostic)
**目的**：找出问题的根本原因
**结构**：问题 → 可能原因 → 子原因 → ... → 根因

```
            为什么产品留存下降？
          /        |         \
    产品问题    市场变化    竞争加剧
   /    \      /    \      /    \
功能 体验   新用户   经济  对手  替代品
bug  差    质量差   下行  降价  出现
```

#### 2. 怎么做树 (How Tree / Solution)
**目的**：找到解决方案
**结构**：目标 → 可能方案 → 子方案 → 行动计划

```
        如何提升用户留存？
      /    |    \    \
   产品   运营   定价  渠道
   优化   策略   调整  优化
```

#### 3. 是否树 (Whether Tree / Decision)
**目的**：做决策
**结构**：决策问题 → 判断标准 → 每个选项评估

### 议题树构建原则
- 每个分支是 MECE 的
- 每层分解 3-5 个分支为宜（太少不够，太多记忆困难）
- 分支到"可操作"或"可验证"为止
- 同一层级使用同一逻辑维度

## 五、麦肯锡七步成诗法

```
① 定义问题 ──→ ② 结构化问题 ──→ ③ 排定优先
                              │
                              ▼
⑥ 综合结果 ←── ⑤ 分析问题 ←── ④ 制定计划
     │
     ▼
⑦ 沟通建议
```

### 各步详解

#### ① 定义问题 (Define the Problem)
- **SMART 原则**：Specific, Measurable, Actionable, Relevant, Time-bound
- **输出**：一句话问题陈述
- **格式**：「在[时间范围]内，[谁/什么]的[指标]从[X]变为[Y]，原因是[初步猜测]」

#### ② 结构化问题 (Structure the Problem)
- 使用逻辑树（Logic Tree）分解
- 选择 MECE 拆解维度
- **输出**：议题树（Issue Tree）

#### ③ 排定优先 (Prioritize)
- 80/20 法则：聚焦影响最大的 20%
- 用矩阵评估：影响 × 可行性
- **输出**：优先级排序的议题清单

#### ④ 制定工作计划 (Plan the Work)
- 每个议题：分析任务 + 负责人 + 截止日 + 数据来源 + 预期产出
- **输出**：工作计划表

#### ⑤ 分析问题 (Analyze)
- 数据收集 → 清洗 → 分析 → 洞察
- 假设驱动的迭代分析
- **输出**：分析发现 (Findings)

#### ⑥ 综合结果 (Synthesize)
- 归纳洞察 → 形成结论 → 提出建议
- 金字塔原理组织
- **输出**：结论+建议摘要

#### ⑦ 沟通建议 (Communicate)
- SCQA 框架讲故事
- 数据图表支持
- 行动建议 + 风险 + 下一步
- **输出**：最终报告/演示

## 六、80/20 法则 (Pareto Principle)

### 核心
20% 的原因导致 80% 的结果。聚焦关键少数。

### 应用
1. 客户分析：20% 客户贡献 80% 收入
2. 问题分析：20% 原因导致 80% 问题
3. 工作量分配：20% 工作产生 80% 价值
4. 时间管理：20% 时间产出 80% 成果

### 实践方法
1. 列出所有因素
2. 量化每个因素的影响
3. 排序，找前 20%
4. 将 80% 精力放在前 20% 上
5. 对剩余 80% 因素做"最小可行处理"

## 七、情景分析 (Scenario Analysis)

### 适用场景
- 未来高度不确定
- 决策影响大、不可逆
- 有多种可能的未来走向

### 方法
1. 识别关键不确定因素（2-3个）
2. 每个因素设定高/低两种可能
3. 交叉形成 2-4 个情景
4. 每个情景设计应对策略
5. 找"无悔行动"（所有情景下都该做的事）

### 情景矩阵示例
```
              经济上行      经济下行
技术突破  │  高速增长   │  技术替代  │
         │  扩产能      │  转型服务  │
─────────┼─────────────┼───────────┤
技术停滞  │  规模扩张   │  收缩防守  │
         │  降本增效   │  核心业务  │
```

## 八、常用分析框架速查

| 框架 | 用途 | 结构 |
|:--|:--|:--|
| **3C** | 战略分析 | Company, Customer, Competitor |
| **4P** | 营销分析 | Product, Price, Place, Promotion |
| **5 Forces** | 行业分析 | Rivalry, New Entrants, Substitutes, Suppliers, Buyers |
| **SWOT** | 态势分析 | Strengths, Weaknesses, Opportunities, Threats |
| **PEST** | 宏观环境 | Political, Economic, Social, Technological |
| **BCG Matrix** | 产品组合 | Market Growth × Market Share |
| **Value Chain** | 运营分析 | Primary Activities + Support Activities |
| **Root Cause (5 Whys)** | 根因分析 | 连问 5 个为什么 |
| **Fishbone (石川图)** | 因果分析 | 人机料法环测 |
| **Decision Matrix** | 决策分析 | 选项 × 标准 × 权重 |
