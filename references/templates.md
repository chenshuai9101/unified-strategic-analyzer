# 麦肯锡分析输出模板

## 使用说明
在分析任何用户任务前，先走一遍这个模板。输出完成后才能进入执行。

---

## 模板：麦肯锡式任务前置分析

### I. 问题定义 (Problem Statement)

用 SMART 原则定义问题：

> **任务**: [用户下达的任务，一句话概括]
> **目标**: [具体可衡量的目标]
> **边界**: [时间/范围/资源的约束条件]
> **成功标准**: [什么算完成任务]

```
例：
任务: 帮用户找 5000 以内拍照最好的手机
目标: 找到一款手机，满足【预算≤5000 + 拍照性能最优】
边界: 预算不可超，覆盖主流品牌（苹果/华为/小米/OPPO/vivo）
成功标准: 输出 1-3 款推荐，附价格+拍照评分+购买建议
```

---

### II. MECE 拆解 (Issue Tree)

把模糊任务拆成清晰的子问题，MECE 原则拆解。

```
任务拆解：

┌─ 1. 需求理解
│   ├─ 1.1 显性需求（用户说了什么）
│   ├─ 1.2 隐性需求（用户没说的）
│   └─ 1.3 约束条件（预算/时间/偏好）
│
├─ 2. 信息搜集
│   ├─ 2.1 数据来源（Web/本地/Agent互联网）
│   ├─ 2.2 数据质量评估
│   └─ 2.3 信息缺口识别
│
├─ 3. 分析判断
│   ├─ 3.1 筛选（按条件过滤）
│   ├─ 3.2 比较（多维度对比）
│   └─ 3.3 推荐（给出最佳选择+理由）
│
├─ 4. 交付执行
│   ├─ 4.1 结果呈现（格式/渠道）
│   └─ 4.2 下一步建议
│
└─ 5. 风险与边界
    ├─ 5.1 已知未知（信息不确定）
    ├─ 5.2 未知未知（可能存在的盲区）
    └─ 5.3 需要用户确认的点
```

---

### III. 初始假设 (Key Hypotheses)

先猜答案，再用数据验证。（假设驱动）

```
假设 1: [最可能的答案]
假设 2: [次可能的答案]  
假设 3: [备选方案]
```

> 每个假设标注置信度：高/中/低

---

### IV. 80/20 聚焦 (Critical Few)

哪些是影响结果的关键少数？哪些可以先放一放？

```
关键路径 (80%影响力的20%工作):
  1. [最关键的一步]
  2. [次要但必须的一步]

可裁剪 (20%影响力的80%工作):
  - [可以简化或跳过的]
  - [对结果影响不大的]
```

---

### V. 分析方案 (Analysis Plan)

| 子任务 | 方法/工具 | 预计时间 | 依赖 | 产出 |
|:--|:--|:--|:--|:--|
| 子任务1 | xx | 2min | 无 | xx |
| 子任务2 | xx | 3min | 子1 | xx |
| ... | ... | ... | ... | ... |

---

### VI. 预期交付物 (Deliverables)

完成分析后，预计输出什么：

```
1. [核心结论] - 一句话回答用户问题
2. [支撑证据] - 关键数据/信息来源
3. [行动建议] - 用户下一步可以做什么
4. [风险提示] - 需要注意的边界和不确定性
```

---

## 简易版模板（日常使用）

对于简单任务，使用精简版：

```
📋 问题定义: [一句话]
🌳 MECE拆解: [2-3个子问题]
💡 关键假设: [最可能的答案]
🎯 关键路径: [最先做什么]
📊 交付: [预计产出什么]
```
