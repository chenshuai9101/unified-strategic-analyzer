## 触发条件

### Quick Mode 触发（任何任务前的第一件事）
- **一定触发**: 用户问"分析/为什么/怎么办/怎么选/要不要"
- **可能触发**: 比较/推荐/决策类问题
- **不触发**: 简单问候/闲聊/纯事实检索

### Deep Mode 触发
- 用户明确要求行业分析/市场研究/竞争格局
- 任务复杂度高且需要系统性报告
- 用户说"帮我做一份XX分析报告"

## 核心方法论

### 5大必用原则（来自互联网学习）

1. **MECE 原则** (Mutually Exclusive, Collectively Exhaustive)
   - 拆解为"不重叠、无遗漏"的子问题
   - AI可以快速生成初始拆解，但人必须检查

2. **假设驱动** (Hypothesis-Driven)
   - 先大胆假设最可能的答案
   - 用AI生成假设 → 人评分排序 → 最可能的下钻验证

3. **80/20 法则** (Pareto Principle)
   - 找出影响最大的20%关键问题
   - AI可以帮排列优先级，但决策权在人

4. **议题树** (Issue Tree)
   - Why Tree：诊断问题（"为什么会这样"）
   - How Tree：制定方案（"怎么做"）

5. **金字塔沟通** (Pyramid Principle)
   - 结论先行 → 二级论据 → 三级数据
   - 顶层结论一句话让听众秒懂

### 新增：AI辅助MECE的5步法
> 来源：High Bridge Academy + 麦肯锡方法论

```
第1步：精确问题定义 → "一句话说清要解决什么"
  AI prompt: "帮我精炼问题陈述，使其具体、可量化、决策导向"

第2步：构建MECE议题树 → AI生成初始分支
  AI prompt: "生成[问题]的MECE拆解，顶层分类+子问题"
  人工检查：分支是否有重叠？是否有遗漏？

第3步：假设生成与排序 → AI生成初步假设
  AI prompt: "对每个分支生成一个可验证的假设"
  人工评分：影响度(1-5) × 验证难度(1-5) → 优先验证高分

第4步：压力测试 → AI挑战你的结构
  AI prompt: "这个结构漏了什么？哪个假设可能是错的？"
  不要接受所有批评，目标是暴露盲区

第5步：金字塔沟通 → AI帮你做Top-Down输出
  AI prompt: "用金字塔原理写摘要——结论先行，论据支撑"
```

## Quick Mode 流程（5步法）

```
[用户任务]
    │
    ▼
① 问题定义 → "一句话说清用户任务是什么"
    │
    ▼
② MECE拆解 → 2层议题树（AI辅助生成+人工检查）
    │
    ▼
③ 关键假设 → 至少1个可验证的假设
    │
    ▼
④ 80/20聚焦 → 影响最大的20%工作
    │
    ▼
⑤ 执行路径 → 推荐的行动顺序
```

### 输出规范（强制）
- **问题定义**: 一句话说清用户问什么
- **MECE拆解**: ≥2层级议题树
- **关键假设**: ≥1个假设+置信度
- **80/20聚焦**: 推荐优先做的1-3件事

## Deep Mode 流程（7步法）

```
[行业分析任务]
    │
    ▼
1️⃣ 问题定义（SMART + SCQA）
    │
    ▼
2️⃣ MECE拆解（议题树 → 研究维度）
    │
    ▼
3️⃣ 初始假设（假设驱动）
    │
    ▼
4️⃣ 80/20聚焦（关键少数）
    │
    ▼
5️⃣ 框架选型（PEST/五力/3C/价值链…）
    │
    ▼
6️⃣ 分析执行（联网搜索 + 数据采集 + 判断）
    │
    ▼
7️⃣ 报告输出（docxtpl专业排版 → .docx）
```

## 使用示例

### Quick Mode 示例
> 用户说："帮我分析为什么Agent互联网没有人用"

```
📋 问题定义: Agent互联网内容平台建成后，
GitHub 仓库仅获~7独立访客/14天，0 star 0 fork，为什么？

🌳 MECE拆解:
  ├─ 1. 内容不足 → 仅18条demo数据，无真实商家
  ├─ 2. 认知不足 → GitHub不推小仓库，社区帖未发
  ├─ 3. 价值不显 → 无让人秒懂的使用场景
  └─ 4. 门槛太高 → 需先启动服务，无在线演示

💡 关键假设: 核心是「认知+价值」问题

🎯 80/20聚焦: 补齐社区帖 > 做在线demo > 拉真实内容
```

### Deep Mode 示例
> 用户说："分析中国ADC药物竞争格局"

```
📋 问题定义: 中国ADC药物竞争格局分析（2026）

🌳 MECE拆解:
  ├─ 已获批药物（现在能卖什么）
  ├─ 在研管线（未来2-3年谁来）
  ├─ 企业地图（谁在布局）
  ├─ 支付准入（谁付钱）
  └─ 趋势建议（入局策略）

💡 关键假设: HER2已红海、TROP2正在变红、CLDN18.2即将内卷

📊 框架选型: 五力模型（竞争烈度）+ 3C（企业地图）+ SWOT

📄 输出: python3 /tmp/md2docx.py adc-report.md output.docx
```

## 压力检查清单（互联网学习的核心补充）

Deep Mode 报告交付前，逐一检查：

- [ ] 问题定义够精确吗？（SMART标准）
- [ ] MECE拆解有重叠分支吗？
- [ ] 每个假设都有数据支撑吗？（不是猜）
- [ ] 80/20聚焦到了最关键的维度？
- [ ] 结论是金字塔结构（结论先行）吗？
- [ ] 数据来源标注了吗？（信源强度）
- [ ] 有没有"被AI训练数据污染"的虚假信息？
